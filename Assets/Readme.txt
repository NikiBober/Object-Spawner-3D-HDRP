Spawner

Для работы спавнера необходимо разместить его в нужном месте на локации.
Объекты для спавна нужно затянуть на конкретный Spawner в окне иерархии (они станут дочерними). 
Можно выбрать для затягивания все объекты за раз.
В параметрах можно определить область спавна.
Те объекты, которые выходят за эту область, автоматически не будут браться для спавна.
Любой объект можно повернуть и расположить необходимым образом, он так и будет появляться.
Также можно выбрать цвет визуализации области спавна для каждого спавнера независимо.
Spawn Cooldown - это задержка до следующего появления предмета, после его поднятия.
Collision Layer - слой, с объектами которого не должна пересекаться область спавна.
Если любой коллайдер с этого слоя попадет в область спавна, предмет не будет спавниться, а в консоль выведется сообщение.

---------------------------------------------
Пару слов об использованных мной решениях:
Использование дочерних объектов вместо списка префабов позволяет увеличить и производительность, и удобство.
Активация и дективация объектов более эффективна чем создание и уничтожение (паттэрн Object pool).
Так на спавнер можно затянуть все объекты, а не по одному на массив в инспекторе (массив будет быстрее списка, если не изменять его при выполнении).
Плюс возможность выставить каждый объект так как нужно, с разным поворотом и положением.
Для визуализации области спавна я использовал гизмо вместо коллайдера.
Так в сцене будет меньше коллайдеров, а гизмо отрисовуются только в редакторе и не оказывают никакого влияния при выполнении.
Также визуализация намного лучше - область полностью заливается цветом, который можно настроить для каждого спавнера отдельно, включая прозрачность (при необходимости можно изменить чтоб рисовались только границы).
Для генерации случайных чисел использовал System.Random.Next вместо Random.Range как более быстрый. 
Так экземпляр класса создается только один раз, а не каждый раз при обращении к методу.
